IMPORTANTE

Ao atualizar o projeto, substitua TODOS os arquivos e pastas, inclusive:

.github/workflows/build-apk.yml
.github/workflows/deploy-supabase.yml

Se a pasta .github antiga permanecer, o GitHub Actions executará o workflow antigo e os testes interromperão o build.

Depois do envio, crie uma NOVA execução pelo novo commit. Não use Re-run jobs em uma execução antiga.
